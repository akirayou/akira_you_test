# VR_DoF_Tester 現状仕様

高校生向けワークショップで、VR における `3DoF` と `6DoF` の見え方の違いを体感するための WebVR アプリです。ブラウザから URL を開いて起動し、同じ空間を複数の視点モードで比較できます。

## 技術構成

- HTML5 / JavaScript
- A-Frame 1.7 系 CDN
- WebXR Device API
- `immersive-vr` セッション対応端末ではゴーグル表示と 6DoF を利用

## 画面構成

### 起動画面

- 中央に「スタート（VR開始）」ボタンを表示
- ワークショップ用途の短い説明文を表示
- 起動時にステータスメッセージ領域を表示

### 操作オーバーレイ

開始後は画面上部と下部にオーバーレイ UI を表示し、以下を操作できます。

- `3DoF 単眼`
- `3DoF ゴーグル`
- `6DoF`
- `視点を戻す`
- 現在モードの説明表示

## 空間演出

- 原点付近から前方へ伸びる細い一本橋を配置
- 橋の足元には支柱を追加
- 下方には簡易ビル群を多数配置して高所感を演出
- 遠景として空色の薄いフォグ層と暗めの空を配置

## 視点仕様

### 共通

- アイレベル基準は `1.4m`
- 初期位置は `0, 1.4, 0`

### `3DoF 単眼`

- 通常の 1 画面表示
- `look-controls` により端末の向き変化を反映
- 位置移動は行わず、視点位置は固定

### `3DoF ゴーグル`

- `immersive-vr` を使った両眼立体視表示
- 回転はヘッドトラッキングに追従
- 実装上は `local-floor` を使い、毎フレーム位置移動を打ち消すことで 3DoF 的な挙動を再現
- 視線カーソルを表示

### `6DoF`

- `immersive-vr` セッションに入る
- 対応端末では位置移動も反映される
- 視線カーソルを表示

## 起動と権限

- iOS / Safari 系を考慮し、必要な場合は `DeviceOrientationEvent.requestPermission()` を呼ぶ
- `navigator.xr.isSessionSupported("immersive-vr")` で対応確認を行う
- 非対応端末では `3DoF 単眼` を中心に利用し、未対応メッセージを表示

## 視点リセット

- `3DoF 単眼` と `3DoF ゴーグル` では、視点を `0, 1.4, 0` 基準へ戻せる
- `6DoF` 中は端末側トラッキングに依存するため、アプリ側では完全なリセットを行わず案内文を表示

## ファイル構成

- `VR_DoF_Tester/index.html`
  UI と A-Frame シーン定義
- `VR_DoF_Tester/app.js`
  モード切り替え、権限処理、視点制御、都市生成ロジック

## 補足

- 大きな外部 3D アセットは使わず、A-Frame のプリミティブで構成
- 配布用ファイルはビルド後に `out/VR_DoF_Tester/` へ生成される
